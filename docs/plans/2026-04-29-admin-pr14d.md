## 2. TDD 컨벤션 (모든 chunk 공통)

14c plan §2 그대로:
1. **Step 1**: 실패 테스트 작성 → `pnpm test:run <path>` FAIL 확인
2. **Step 2**: 최소 구현 → 동일 명령 PASS
3. **Step 3**: typecheck (`pnpm exec tsc --noEmit`) + commit

대규모 chunk(G2/G3/G4)는 chunk 안 task 단위로 commit. atomic group(G<n>) + polish follow-up(G<n>.1 등) 메모리 패턴 일관.

**테스트 idiom**:
- `userEvent.setup({ pointerEventsCheck: 0, delay: null })` (jsdom radix)
- shadcn Select/Checkbox: `getByRole("combobox")` / `getByRole("checkbox")` + `aria-label` 필수
- Dialog 내부 Select user.click은 jsdom hang(14c §14 entry 14) — 회피 패턴: controlled prop / re-render
- `vi.spyOn(toast, "...")`로 toast assertion
- mutation hook: `renderHook` + `QueryClientProvider` wrapper + `waitFor(() => expect(result.current.isSuccess).toBe(true))`

  ```ts
  export function useBulkPartyroomAction() {
    const qc = useQueryClient()
    return useMutation({
      mutationFn: (body: BulkPartyroomActionRequest) => bulkPartyroomAction(body),
      onSuccess: (response) => {
        qc.invalidateQueries({ queryKey: ["partyrooms"] })
        const ok = response.results.filter(r => r.success).length
        const ng = response.results.length - ok
        if (ng === 0) toast.success(`일괄 처리 완료 (${ok}건)`)
        else if (ok === 0) toast.error("일괄 처리 전건 실패")
        else toast.warning(`성공 ${ok}건 / 실패 ${ng}건`)
      },
      onError: mutationErrorToast,
    })
  }
  ```
